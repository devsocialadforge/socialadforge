# Project Rules

## 1. Code Style
- Use clean, readable code.
- Use Tailwind for styling.
- Use shadcn/ui components.
- Use GSAP for animation.
- Follow SEO best practices.

## 2. File Structure
- Keep components inside `/components`
- Use `/lib` for helpers
- Use `/app` for pages (Next.js)

## 3. Commit Rules
- Write clear commit messages.
- Format: `type(scope): short message`

## 4. Response Rules for AI
- Always show code only when required.
- Prefer TypeScript.
- Do not change existing logic unless asked.
- Explain changes briefly.

## 5. UI/UX Rules
- Use minimalistic, modern design.
- Keep consistent spacing and responsive layout.
- Follow project brand colors and font for UI.
- Keep font sizes, weights, and colors consistent.


## 6. Responsiveness Rules
- Design must be responsive across **mobile**, **tablet**, and **desktop**.
- Maintain consistent:
  - **Font size hierarchy**
  - **Padding and margin spacing**
  - **Font weight usage**
- Use Tailwind responsive utilities (`sm:`, `md:`, `lg:`) for layout and typography.
